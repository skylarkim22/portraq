# PRD 동기화 검증

현재 프로젝트 코드베이스와 `docs/PRD.md`의 내용이 일치하는지 검증합니다.

## 검증 절차

다음 순서대로 실행하세요.

### 1단계 — 현재 상태 수집

아래 파일들을 읽어 실제 프로젝트 상태를 파악합니다:
- `package.json` — 실제 설치된 의존성과 버전
- `next.config.*` — Next.js 설정 (존재할 경우)
- `tsconfig.json` — TypeScript 설정 (존재할 경우)
- `tailwind.config.*` — Tailwind 설정 (존재할 경우)
- `docs/PRD.md` — 현재 PRD 문서

### 2단계 — 비교 및 불일치 탐지

`package.json`의 실제 의존성과 PRD 4.1 섹션 기술 스택 표를 비교합니다.
다음 항목을 검사합니다:
- 라이브러리 추가/제거 여부 (PRD에 없는 패키지, PRD에만 있는 패키지)
- 버전 불일치 (major 버전 차이, 또는 "추후 설치 예정"이지만 이미 설치된 경우)
- 패키지 매니저 변경 여부 (`packageManager` 필드)

그 외 PRD에 기술된 기능/화면 중 코드에서 확인 가능한 사항도 점검합니다:
- `src/app/` 디렉토리 구조와 PRD 3.1 주요 화면 목록 비교

### 3단계 — 결과 보고

검증 결과를 아래 형식으로 출력합니다:

```
✅ 일치 항목: ...
⚠️  불일치 항목:
  - [항목명] PRD: "..." / 실제: "..."
  - ...
```

불일치 항목이 없으면 "PRD와 코드베이스가 일치합니다." 메시지를 출력하고 종료합니다.

### 4단계 — 업데이트 여부 확인

불일치 항목이 있을 경우, 사용자에게 아래와 같이 묻습니다:

> 위 불일치 항목을 `docs/PRD.md`에 반영할까요? (y/n)

- **y**: PRD의 해당 섹션을 실제 코드베이스 기준으로 업데이트합니다.
- **n**: 변경 없이 종료합니다. 의도적인 차이라면 PRD에 별도 메모를 남길지 추가로 묻습니다.
