## 1. 서비스 개요

### 1.1 서비스 비전

Portraq는 두 가지 핵심 서비스를 제공합니다.

첫째, 개인 투자자가 매월 일정 금액을 분할 투자하는 적립식 투자(Dollar-Cost Averaging)를 체계적으로 실행할 수 있도록 돕는 **포트폴리오 관리 서비스**입니다. 검증된 대가의 전략을 템플릿으로 제공하고, 현재 보유 현황을 반영한 리밸런싱 기반 매수·매도 가이드를 통해 매달 투자 실행을 명확하게 안내합니다.

둘째, 실제 매수·매도를 실행한 날 이유와 수량·가격·세금을 기록하는 **매매 일지 서비스**입니다. 거래를 기록하고 돌아보는 습관을 통해 즉흥적인 의사결정을 방지하고, 자신의 투자 패턴을 객관적으로 파악할 수 있도록 돕습니다. 두 서비스는 완전히 독립적으로 동작하며, 각각 별도로 활용할 수 있습니다.

### 1.2 핵심 문제 정의

**포트폴리오 관리:**
- 적립식 투자를 하고 싶지만 어떤 종목에 얼마를 배분해야 할지 모르는 개인 투자자
- 검증된 대가들의 전략을 따르고 싶지만 직접 종목을 조사하기 어려운 투자자
- 포트폴리오를 관리하는 별도 도구 없이 엑셀이나 메모장에 기록하는 비효율
- 매수·매도 결정을 내린 이유를 체계적으로 남기지 못해 과거 판단을 복기하기 어려운 투자자

**매매 일지:**
- 매수·매도 시 충동적으로 결정하고 이유를 기록하지 않아 같은 실수를 반복하는 투자자
- 자신이 언제, 얼마에, 왜 샀는지 돌아볼 수단이 없어 투자 패턴을 파악하지 못하는 투자자
- 세금·손익 계산을 따로 정리하지 않아 실제 수익률을 모르는 투자자

### 1.3 타겟 사용자

| 항목 | 내용 |
|------|------|
| **주요 대상** | 주식 투자 경험이 1년 미만~3년의 개인 투자자, 적립식 투자에 관심 있는 직장인 |

## 2. 핵심 기능 명세

### 2.1 대가 포트폴리오 템플릿

투자의 대가들이 실제 공개한 철학 및 포트폴리오를 기반으로 사전 구성된 템플릿을 제공합니다. 사용자는 이를 그대로 사용하거나 자신의 입맛에 맞게 수정할 수 있습니다.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 템플릿 선택 | 워런 버핏, 레이 달리오(올웨더), 캐시 우드, 마이클 버리 등 국내외 대가 포트폴리오 제공 | P0 |
| 템플릿 미리보기 | 카드 탭 시 슬라이드다운 상세 패널 — 투자 철학, 종목 구성 바차트, 연평균 수익률(CAGR), 최대낙폭(MDD) 표시 | P0 |
| 템플릿 기반 편집 | 템플릿 불러온 후 종목 추가/제거/비율 수정 가능 | P0 |
| 미확정 슬롯 안내 | 기타 종목처럼 비중은 있으나 종목이 특정되지 않은 슬롯을 별도 표시, 사용자가 직접 채우도록 유도 | P0 |
| 템플릿 즐겨찾기 | 자주 참고하는 템플릿 북마크 기능 | P2 |

#### 2.1.1 갤러리 UI 상세

- **레이아웃**: 카드 그리드 (모바일 2열 / 데스크탑 3열), 반응형 대응
- **카드 구성 요소**:
  - 대가 이름 및 전략명
  - 전략 방식 배지 (패시브 / 가치투자 / 퀀트·팩터 / 자산배분 / 성장)
  - 시장 배지 (🇰🇷 KR / 🇺🇸 US)
  - 10년 연평균 수익률 (CAGR)
  - 최대낙폭 (MDD) — 수익률과 나란히 표시해 리스크를 수치로 직접 비교
- **상세 패널** (카드 탭 시 슬라이드다운, 페이지 이동 없음):
  - 투자 철학 요약 2~3줄
  - 연평균 수익률(CAGR) + 최대낙폭(MDD) 수치 강조 표시
  - MDD 바 시각화 — 낙폭 크기를 직관적으로 비교
  - 종목 구성 가로 바 차트 (비중 시각화)
  - "이 포트폴리오 사용하기" CTA → 편집 화면 진입
  - "직접 구성" 버튼 → 빈 포트폴리오로 진입 (갤러리 우상단)
- **미확정 슬롯 처리**:
  - 버핏의 기타(비공개 종목) 20.1%, 우드의 기타 ARK 보유 종목 57.1%처럼 공개 포트폴리오에서 특정되지 않은 종목은 미확정 슬롯으로 표시
  - 편집 화면에서 슬롯은 + 아이콘과 '클릭해서 종목을 추가하세요' 안내 문구가 있는 점선 카드로 표시
  - 슬롯 클릭 시 검색 모달이 열림 → 종목 선택 시 기존 비중·순서를 유지한 채 슬롯이 실제 종목으로 교체
  - 미확정 슬롯이 남아 있으면 저장 시 경고 토스트 표시 (강제 차단은 하지 않음)
- **필터 탭**: 전체 / 패시브 / 가치투자 / 퀀트·팩터 / 자산배분

> ※ 주관적 리스크 라벨(안정형/중위험/성장형) 대신 전략 방식으로 분류. 리스크 수준은 MDD 수치로 직접 판단.

**제공 대가 포트폴리오:**

- 워런 버핏 – 집중 투자, 우량주 중심 (AAPL, AXP, KO 등, 2026 Q1 13F 기준)
- 레이 달리오 – 올웨더 포트폴리오 (주식 30%, 장기채 40%, 금 7.5% 등)
- 캐시 우드 – 파괴적 혁신 성장주 집중 투자 (ARK Invest ARKK 기준, TSLA·AI·로보틱스 등)
- 마이클 버리 – 역발상 가치투자, "빅쇼트"로 유명 (Scion Asset Management 13F 기준)
- 존 보글 – 인덱스 펀드 중심 패시브 투자 (ETF 집중)
- 사용자 직접 구성 – 빈 포트폴리오에서 시작

#### 2.1.2 대가별 전략 분류 및 수익·리스크 지표

주관적 리스크 라벨(안정형·중위험·성장형)을 사용하지 않고, 전략 방식과 정량 지표(CAGR, MDD)로 분류합니다.

| 대가 | 전략 방식 | 시장 | CAGR (10Y) | MDD |
|------|-----------|------|------------|-----|
| 레이 달리오 | 자산배분 | US | +7.4% | -20.6% |
| 존 보글 | 패시브 | US | +9.8% | -21.0% |
| 워런 버핏 | 가치투자 | US | +10.4% | -32.7% |
| 캐시 우드 | 성장 | US | +13.5% | -80.9% |
| 마이클 버리 | 가치투자 | US | +26.7% | - |

> ※ CAGR과 MDD는 백테스트·공개 데이터 기반 참고값으로, 실제 수익률과 다를 수 있습니다. 특히 마이클 버리는 13F에 롱 주식·옵션만 공시되고 공매도·해외 자산은 드러나지 않아 실제 전략의 일부만 반영합니다. 투자 결정의 참고 자료로만 활용하세요.

### 2.5 리밸런싱 기반 매수·매도 가이드

매달 투자 실행 시 현재 보유 현황을 반영해 종목별 매수/매도/유지 액션을 정확히 안내합니다.

#### 2.5.1 3단계 입력 흐름

**Step 1 — 현재 보유 주수 입력:**
- 종목별 보유 주수를 직접 타이핑. 보유하지 않은 종목은 0으로 두면 됨
- 카드 UI: 티커(색상 구분) + 종목명 + 목표 비율 표시 + 보유 주수 입력 필드

**Step 2 — 투자금·환율·현재가 입력:**
- 이번 달 추가 투자금 입력 (10만/30만/50만/100만 프리셋 버튼 제공)
- 매도 임계값(%p) 입력 — 목표 비율과의 괴리가 이 값 미만인 초과 비중 종목은 매도 없이 유지. 기본값 5%p, 사용자가 직접 조정 가능
- **원/달러 환율 입력**: 미국 주식 현재가를 원화로 환산하는 데 사용. 실시간 조회가 아닌 사용자 직접 입력값이며, 매매 시점의 실제 환율을 별도로 확인해 입력하도록 안내 문구 표시 (7.3의 "환율 자동 반영"은 이 수동 입력을 실시간 API 조회로 대체하는 향후 확장이며, 이번 스텝의 수동 입력 UI와는 별개)
- 종목별 현재가 입력:
  - 한국 주식: 원(step 100) 직접 입력
  - 미국 주식: 달러(step 0.01) 입력 → 입력한 환율을 적용한 원화 환산액을 종목 카드에 실시간 표시
  - 현재가 기본값: 해당 종목의 가장 최근 저장된 스냅샷 가격(원화 환산 기준)을 자동으로 채움. 저장 이력이 없으면 0으로 세팅. 사용자가 직접 수정 가능
- 보유 주수 × 현재가(원화 환산) = 보유 금액이 실시간 계산되어 표시, 전체 보유 금액 기준 현재 비율 자동 산출
- 현재 총 보유 금액 + 추가 투자금 = 운용 후 총액 실시간 표시

**Step 3 — 종목별 액션 확인 및 수정:**
- 미확정 슬롯이 남아 있으면 경고 배너 표시 + Step 3 결과 숨김. 모든 슬롯을 채워야 결과 표시
- 운용 후 총액 × 목표 비율 = 종목별 목표 금액 계산
- 목표 금액 - 현재 보유 금액 = 차이 → 매수/매도/유지 자동 결정
- 차이가 1주 미만이면 유지로 처리 (불필요한 소액 거래 방지)
- **매수 우선, 매도 최소화**: 추가 투자금은 저비중 종목 매수에 우선 사용된다. 초과 비중 종목은 목표 비율과의 괴리(%p)가 Step 2에서 설정한 매도 임계값 이상일 때만 매도로 계산하고, 임계값 미만이면 매도 없이 유지한다 (매수 판단에는 임계값을 적용하지 않음)
- 자동 계산된 액션(매수/매도 주수)은 사용자가 직접 수정 가능 — 수정 시 해당 종목 금액 및 합계 실시간 재계산
- 저장 시 사용자가 최종 확정한 값을 ExecutionRecord에 기록
- 매도 발생 시 안내 문구: 매도 회수 금액을 투자금과 합산해 매수에 활용

#### 2.5.2 액션 유형

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 매수 | 목표 금액 > 현재 보유 금액 — 매수 주수 + 금액 자동 계산, 사용자 수정 가능 | P0 |
| 매도 | 목표 금액 < 현재 보유 금액이고 괴리가 매도 임계값 이상 — 매도 주수 + 금액 자동 계산, 사용자 수정 가능 | P0 |
| 유지 | 차이가 1주 미만(한국·미국 공통), 또는 초과 비중이지만 괴리가 매도 임계값 미만 — 거래 불필요. 사용자가 직접 매수/매도로 변경 가능 | P0 |
| 매도 임계값 조정 | Step 2에서 매도 임계값(%p)을 직접 입력해 매도 민감도 조정 (기본값 5%p) | P0 |
| 현재 비율 표시 | 현재 비율 vs 목표 비율 괴리(%p)를 색상으로 구분 표시 | P0 |
| 매도 후 안내 | 매도 종목 발생 시 회수 금액 합산 안내 문구 표시 | P0 |
| 리밸런싱 알림 | 목표 비율 괴리 5%p 이상 시 홈 화면에 알림 배지 표시 | P1 |

### 2.6 매매 일지

포트폴리오 리밸런싱 서비스와 완전히 독립된 별도 서비스입니다. 실제 매수·매도를 실행한 날 이유와 수량·가격·세금을 기록하는 투자 일지 기능으로, 매매 일지 자체 데이터만을 기반으로 동작합니다.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 매수 기록 | 날짜 + 종목(복수) + 수량·가격 + 메모 입력. 종목은 검색으로 자유 추가 | P1 |
| 매도 기록 | 매매 일지 내 매수 기록 기반 보유 종목 목록에서 선택 → 수량·가격·세금·환율 입력 | P1 |
| 달력 뷰 | 월별 달력에 매수(초록)·매도(빨강) 점 표시. 날짜 탭 시 해당일 일지 피드 | P1 |
| 월별 통계 | 총 매수금액·총 매도금액·세금 합계·순손익·순수익률·거래 횟수·시장별 비중·많이 거래한 종목 Top3 표시 | P1 |
| 평균단가 자동 계산 | 매매 일지 내 누적 매수 기록만으로 종목별 평균단가·보유 수량을 동적 산출. 매도 화면에서 참조 | P1 |

#### 2.6.1 매수 기록 작성

- 날짜 (기본값: 오늘, 수정 가능)
- 종목 카드 (복수 추가 가능): 티커·종목명 검색 → 수량·가격 입력 → 합계(수량 × 가격) 자동 표시
- 메모 (선택, 최대 1,000자)

#### 2.6.2 매도 기록 작성

매도는 기존 매수 기록에서 파생된 보유 종목만 선택 가능합니다.

- 날짜 (기본값: 오늘, 수정 가능)
- 보유 종목 목록: 매수 기록 기반으로 종목명·보유 수량·평균단가 표시 → [+ 추가] 탭으로 매도 대상에 추가
- 매도 종목 카드: 수량 (보유 수량 초과 불가) · 가격 입력
  - US 종목: 환율(원/$) 추가 입력 → 매도금액 원화 환산 자동 계산
  - KR 종목: 환율 입력 불필요
  - 세금 (원): 직접 입력 (자동 계산 없음 — 증권사·공제 항목별 상이)
  - 매도금액 · 손익(매도가 − 평균단가 × 수량) · 세후 손익(손익 − 세금) 실시간 표시
- 메모 (선택, 최대 1,000자)

**손익 계산 방식:** 평균단가법 적용. 매수 기록 수정 없이 동적 산출.

```
평균단가 = Σ(매수 수량 × 매수가) ÷ 총 보유 수량
손익     = (매도가 − 평균단가) × 매도 수량
세후 손익 = 손익(원 환산) − 세금
```

#### 2.6.3 달력 뷰 및 월별 통계

**달력:**
- 월별 달력, 매수일 초록 점·매도일 빨간 점·혼재일 두 점 표시
- 날짜 탭 → 해당일 일지 카드 피드 (종목 배지·합계·메모 미리보기)

**월별 통계 (달력 하단 고정):**

| 지표 | 계산 기준 |
|------|----------|
| 총 매수금액 | 해당 월 매수 거래 합계 (원 환산) |
| 총 매도금액 | 해당 월 매도 거래 합계 (원 환산) |
| 세금 합계 | 해당 월 입력된 세금 합계 |
| 순손익 | Σ(매도가 − 평균단가) × 매도 수량 − 세금 합계 (원 환산) |
| 순수익률 | 순손익 ÷ 매도분 매수원금 × 100 (%) |
| 거래 횟수 | 해당 월 매수 + 매도 기록 건수 |
| 시장별 비중 | 해당 월 매수금액 기준 KR·US 비율 (가로 바 시각화) |
| 많이 거래한 종목 Top 3 | 거래 횟수 기준, 매수·매도 건수 구분 표시 |

**하단 버튼:** 매수 기록 / 매도 기록 버튼을 나란히 배치해 해당 작성 화면으로 직접 진입

- `ticker`: string — 종목 코드
- `action`: buy | sell | hold — 액션 유형
- `quantity`: number — 매수(+) 또는 매도(-) 주수
- `pricePerShare`: number — 실행 시점 주당 가격 (원화 환산 기준. 미국 주식은 Step 2에서 입력한 환율로 환산된 값)
- `totalAmount`: number — quantity × pricePerShare (매도는 음수)

### 5.4 PortfolioSnapshot (포트폴리오 스냅샷)

저장 버튼 클릭 시마다 생성되는 해당 시점의 포트폴리오 전체 상태입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string (UUID) | 스냅샷 고유 식별자 |
| portfolioId | string | 연결된 포트폴리오 ID |
| executionRecordId | string | 연결된 실행 기록 ID (1:1 대응) |
| assets | SnapshotAsset[] | 해당 시점 전체 종목 구성 배열 |

**SnapshotAsset 구조:**
- `ticker`: string — 종목 코드
- `name`: string — 종목명
- `ratio`: number — 해당 시점 목표 비중 (%)
- `shares`: number — 해당 시점 보유 주수
- `pricePerShare`: number — 해당 시점 현재가 (원화 환산 기준)
- `color`: string (hex) — 종목 고정 색상

### 5.5 저장 시 처리 흐름

"저장"은 화면에 따라 서로 다른 두 가지 이벤트를 의미하며, 처리 범위가 다릅니다.

**① 포트폴리오 편집 화면의 저장** — 목표 비중·종목 구성·이름/메모를 조정하는 이벤트입니다.
- **PortfolioAsset 업데이트** — 현재 종목 구성·목표 비중 덮어쓰기 (이름/메모 포함)
- ExecutionRecord·PortfolioSnapshot은 생성하지 않습니다. 실제 매수·매도가 일어나지 않았는데 실행 기록을 남기면 이력이 의미 없는 데이터로 오염되기 때문입니다.

**② 리밸런싱 가이드의 최종 확정 저장** — Step 1~3을 거쳐 실제 보유 현황과 이번 달 매수·매도 액션을 확정하는 이벤트입니다. 아래 세 가지가 하나의 트랜잭션으로 원자적으로 처리됩니다:

1. **PortfolioAsset 업데이트** — 확정된 보유 주수·현재가로 갱신 (다음 리밸런싱의 시작점이 됨)
2. **PortfolioSnapshot 생성** — 확정 시점의 전체 상태 스냅샷 신규 레코드 삽입
3. **ExecutionRecord 생성** — 매수·매도·유지 액션 및 금액 신규 레코드 삽입

> ※ 위 세 작업은 DB 트랜잭션으로 묶어 처리합니다. 하나라도 실패하면 전체 롤백.

### 5.6 종목 색상 시스템

종목마다 고정된 색상을 부여해 편집 화면, 미니 비중 바, 매수·매도 가이드 등 모든 화면에서 동일한 색으로 표시합니다.

#### 5.6.1 티커 해시 기반 색상 결정

Asset 마스터 테이블에 종목을 최초 insert할 때 색상을 계산해 함께 저장합니다. 이후에는 재계산 없이 Asset.color를 그대로 참조합니다.

```javascript
function getTickerColor(ticker, palette) {
  const hash = ticker.split('').reduce((acc, c) => acc + c.charCodeAt(0), 0);
  return palette[hash % palette.length];
}
```

#### 5.6.2 팔레트 구성

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 팔레트 크기 | 20개 색상 — 15종목 이하 포트폴리오에서 겹침 거의 없음 | P0 |
| 색상 간격 | 인접 색상 간 색조(hue) 차이 18° 이상 확보 | P0 |
| 충돌 감지 | 같은 포트폴리오 내 동일 색 발생 시 팔레트에서 다음 색으로 순환 배정 | P0 |
| 색상 저장 | 종목 최초 등록 시 확정된 색상을 Asset.color 필드에 저장. 이후 모든 화면에서 공통 참조 | P0 |
| 색상 고정 | 저장 이후에는 티커가 동일해도 저장된 색상을 우선 사용 (재계산 없음) | P0 |
| 색맹 대응 | 색상만으로 구분하지 않고 티커 텍스트를 항상 함께 표시 (색상은 보조 수단) | P1 |

#### 5.6.3 색상 적용 범위

- 포트폴리오 편집 화면 — 종목 카드 티커 텍스트 색상 및 비중 바
- 내 포트폴리오 목록 — 카드 내 미니 비중 바 세그먼트
- 이달의 매수·매도 가이드 — 액션 테이블 종목 티커 색상
- 보유 현황 입력 카드 — 티커 텍스트 및 현재 비율 바

### 5.7 수익 계산 방식

#### 5.7.1 조회 기간 필터

전체 이력 조회 시 DB 부하를 방지하기 위해 기간 단위를 선택해 계산 범위를 제한합니다.

| 단위 | 설명 |
|------|------|
| 일 | 오늘 하루 기준 |
| 주 | 최근 7일 기준 |
| 월 | 최근 30일 기준 |
| 년 | 최근 1년 기준 |

기간 필터는 실현손익·수익률 추이·거래 내역에 적용됩니다. 평가손익은 현재 시점 기준이므로 기간 필터 미적용.

#### 5.7.2 평가손익 (미실현손익)

현재 보유 자산의 미실현 손익입니다. 기간 필터와 무관하게 항상 현재 시점 기준으로 계산합니다.

**계산식:** `(현재가 - 평균 매수가) × 보유 주수`

- 현재가: 최신 PortfolioSnapshot의 pricePerShare
- 평균 매수가: 전체 기간 ExecutionRecord 매수 기록 누적 평균

**예시:** 삼성전자 평균 매수가 70,000원, 현재가 82,000원, 보유 10주  
→ 평가손익 = (82,000 - 70,000) × 10 = **+120,000원**

#### 5.7.3 실현손익

선택한 기간 내 ExecutionRecord 매도 기록 기준으로 계산합니다. 매매 일지와는 완전히 별개의 데이터 소스를 사용합니다.

- **계산식:** `(매도가 - 평균 매수가) × 매도 주수`
- 평균 매수가: 전체 기간 해당 종목의 ExecutionRecord 매수 기록 누적 평균 (기간 필터 미적용)
- 매수 거래는 실현손익 없음 (평균 매수가 산출에만 반영)

#### 5.7.4 표시 지표

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 현재 평가금액 | 보유 주수 × 현재가 합산 (최신 스냅샷 기준) | P2 |
| 총 평가손익 | Σ(현재가 - 평균 매수가) × 보유 주수 | P2 |
| 총 실현손익 | 선택 기간 내 매도 거래 손익 합산 | P2 |
| 총 수익률 | 총 평가손익 ÷ 총 투자 원금 × 100 (%) | P2 |
| 종목별 수익률 | 종목별 (현재가 - 평균 매수가) ÷ 평균 매수가 × 100 (%) | P2 |
| 수익률 추이 | 선택 기간 내 스냅샷 기준 총 평가금액 변화를 꺾은선 그래프로 표시 | P2 |
| 포트폴리오 간 비교 | 복수 포트폴리오의 수익률을 동일 그래프에서 비교 | P2 |

> ※ 수익 계산은 참고용이며 세금·수수료는 미반영입니다. 실제 손익과 다를 수 있습니다.

### 5.8 TradeLog (매매 일지)

매수 또는 매도 기록 1건을 나타냅니다. 포트폴리오 서비스와 완전히 독립된 데이터입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string (UUID) | 고유 식별자 |
| userId | string | Supabase 사용자 ID |
| type | buy \| sell | 매수 또는 매도 구분 |
| date | date | 거래 날짜 (기본값: 오늘, 수정 가능) |
| items | TradeItem[] | 거래 종목 목록 (JSONB) |
| memo | string \| null | 메모 (최대 1,000자) |
| createdAt | timestamp | 기록 생성 일시 |

**TradeItem 구조:**
- `ticker`: string — 종목 코드 (FK → Asset)
- `quantity`: number — 수량
- `price`: number — 매수가 또는 매도가
- `tax`: number | null — 세금 (매도 시만 입력)
- `exchangeRate`: number | null — 환율 원/$ (US 종목 매도 시만 입력)

## 7. 추가 제안 기능

### 7.1 투자 성향 진단 (온보딩)

신규 사용자 진입 시 5~7개 질문으로 투자 성향을 파악하고 적합한 템플릿을 자동 추천합니다. 질문 결과를 MDD 허용 범위와 매핑해 전략 방식을 추천합니다.

> 예: MDD 10% 이하 선호 → 자산배분 전략, 수익률 극대화 선호 → 가치투자 / 퀀트·팩터 전략

### 7.2 시뮬레이션 – 과거 성과 백테스트

선택한 포트폴리오를 과거 N년 동안 매달 적립했다면 수익이 얼마였는지 시뮬레이션으로 보여줍니다.

### 7.3 환율 자동 반영

한국 사용자가 미국 주식 투자 시 원/달러 환율을 반영하여 원화 기준 투자 금액과 매수 수량을 계산합니다.

### 7.4 포트폴리오 공유

자신이 구성한 포트폴리오를 링크로 공유하거나, 커뮤니티에 공개할 수 있는 기능입니다.

### 7.5 증권사 연동 (장기 로드맵)

KIS(한국투자증권) 오픈API 또는 eTrade API를 통해 실제 매수까지 앱 내에서 처리하는 기능입니다.
