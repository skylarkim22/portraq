# 📊 Portraq
## 적립식 투자 포트폴리오 관리 서비스
### Product Requirements Document (PRD) v1.1
> 2025년 6월 (v1.0)

---

## 1. 서비스 개요

### 1.1 서비스 비전

Portraq는 개인 투자자가 매월 일정 금액을 분할 투자하는 적립식 투자(Dollar-Cost Averaging)를 체계적으로 실행할 수 있도록 돕는 포트폴리오 관리 서비스입니다.

### 1.2 핵심 문제 정의

- 적립식 투자를 하고 싶지만 어떤 종목에 얼마를 배분해야 할지 모르는 개인 투자자
- 검증된 대가들의 전략을 따르고 싶지만 직접 종목을 조사하기 어려운 투자자
- 포트폴리오를 관리하는 별도 도구 없이 엑셀이나 메모장에 기록하는 비효율

### 1.3 타겟 사용자

| 항목 | 내용 |
|------|------|
| **주요 대상** | 주식 투자 경험이 1년 미만~3년의 개인 투자자, 적립식 투자에 관심 있는 직장인 |
| **투자 시장** | 한국 주식(KOSPI/KOSDAQ), 미국 주식(NYSE/NASDAQ), 국내외 ETF |
| **투자 성향** | 장기 분산투자 선호, 매월 10만원~100만원 규모 적립 |

## 2. 핵심 기능 명세

### 2.1 대가 포트폴리오 템플릿

투자의 대가들이 실제 공개한 철학 및 포트폴리오를 기반으로 사전 구성된 템플릿을 제공합니다. 사용자는 이를 그대로 사용하거나 자신의 입맛에 맞게 수정할 수 있습니다.

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 템플릿 선택 | 워런 버핏, 레이 달리오(올웨더), 피터 린치, 강환국 등 국내외 대가 포트폴리오 제공 | P0 |
| 템플릿 미리보기 | 카드 탭 시 슬라이드다운 상세 패널 — 투자 철학, 종목 구성 바차트, 연평균 수익률(CAGR), 최대낙폭(MDD) 표시 | P0 |
| 템플릿 기반 편집 | 템플릿 불러온 후 종목 추가/제거/비율 수정 가능 | P0 |
| 미확정 슬롯 안내 | 기타 종목처럼 비중은 있으나 종목이 특정되지 않은 슬롯을 별도 표시, 사용자가 직접 채우도록 유도 | P0 |
| 템플릿 즐겨찾기 | 자주 참고하는 템플릿 북마크 기능 | P2 |

#### 2.1.1 갤러리 UI 상세

- **레이아웃**: 카드 그리드 (모바일 2열 / 데스크탑 3열), 반응형 대응
- **카드 구성 요소**:
  - 대가 이름 및 전략명
  - 전략 방식 배지 (패시브 / 가치투자 / 퀀트·팩터 / 자산배분)
  - 시장 배지 (🇰🇷 KR / 🇺🇸 US / ETF)
  - 10년 연평균 수익률 (CAGR)
  - 최대낙폭 (MDD) — 수익률과 나란히 표시해 리스크를 수치로 직접 비교
- **상세 패널** (카드 탭 시 슬라이드다운, 페이지 이동 없음):
  - 투자 철학 요약 2~3줄
  - 연평균 수익률(CAGR) + 최대낙폭(MDD) 수치 강조 표시
  - MDD 바 시각화 — 낙폭 크기를 직관적으로 비교
  - 종목 구성 가로 바 차트 (비중 시각화)
  - "이 포트폴리오 사용하기" CTA → 편집 화면 진입
  - "직접 구성" 버튼 → 빈 포트폴리오로 진입 (갤러리 우상단)
- **미확정 슬롯 처리**:
  - 버핏의 기타 28%처럼 공개 포트폴리오에서 특정되지 않은 종목은 미확정 슬롯으로 표시
  - 슬롯 카드에 + 아이콘과 '종목을 직접 추가하세요' 안내 문구 표시
  - 슬롯 클릭 시 검색창 자동 포커스 → 종목 선택 시 슬롯이 실제 종목으로 교체
  - 미확정 슬롯이 남아 있으면 저장 시 경고 토스트 표시 (강제 차단은 하지 않음)
- **필터 탭**: 전체 / 패시브 / 가치투자 / 퀀트·팩터 / 자산배분

> ※ 주관적 리스크 라벨(안정형/중위험/성장형) 대신 전략 방식으로 분류. 리스크 수준은 MDD 수치로 직접 판단.

**제공 예정 대가 포트폴리오:**

- 워런 버핏 – 집중 투자, 우량주 중심 (BRK, AAPL, BAC 등)
- 레이 달리오 – 올웨더 포트폴리오 (주식 30%, 장기채 40%, 금 7.5% 등)
- 피터 린치 – 성장주 중심, 아는 기업에 투자
- 강환국 – 퀀트 기반 팩터 투자 전략 (한국 시장 특화)
- 존 보글 – 인덱스 펀드 중심 패시브 투자 (ETF 집중)
- 사용자 직접 구성 – 빈 포트폴리오에서 시작

#### 2.1.2 대가별 전략 분류 및 수익·리스크 지표

주관적 리스크 라벨(안정형·중위험·성장형)을 사용하지 않고, 전략 방식과 정량 지표(CAGR, MDD)로 분류합니다.

| 대가 | 전략 방식 | 시장 | CAGR (10Y) | MDD |
|------|-----------|------|------------|-----|
| 레이 달리오 | 자산배분 | US / ETF | +7.2% | -12.4% |
| 존 보글 | 패시브 | US / ETF | +9.8% | -21.0% |
| 워런 버핏 | 가치투자 | US | +10.4% | -32.7% |
| 피터 린치 | 가치투자 | US / KR | +11.3% | -28.1% |
| 강환국 | 퀀트·팩터 | KR / US / ETF | +12.1% | -22.3% |

> ※ CAGR과 MDD는 백테스트 기반 참고값으로, 실제 수익률과 다를 수 있습니다. 투자 결정의 참고 자료로만 활용하세요.

### 2.5 리밸런싱 기반 매수·매도 가이드

매달 투자 실행 시 현재 보유 현황을 반영해 종목별 매수/매도/유지 액션을 정확히 안내합니다.

#### 2.5.1 3단계 입력 흐름

**Step 1 — 현재 보유 주수·현재가 입력:**
- 종목별 보유 주수를 직접 타이핑
- 현재가는 기본값(등록 시 가격)이 자동 채워지며, 실시간 가격으로 수정 가능
- 보유 주수 × 현재가 = 보유 금액이 실시간 계산되어 카드에 표시
- 전체 보유 금액 기준 현재 비율 자동 산출

**Step 1 입력 카드 UI 규격:**
- 카드 상단: 티커(색상 구분) + 시장 배지(KR/US/ETF) + 종목 전체명 + 우측 보유 금액
- 카드 하단: 보유 주수 입력 필드 / 현재가 입력 필드 — 2열 나란히 배치
- 현재가 단위: 한국 주식은 원(step 100), 미국·ETF는 달러(step 0.01)
- 카드 최하단: 현재 비율 바 (전체 대비 보유 비중 시각화)

**Step 2 — 이번 달 추가 투자금 입력:**
- 현재 총 자산 + 추가 투자금 = 운용 후 총액 실시간 표시

**Step 3 — 종목별 액션 확인:**
- 운용 후 총액 × 목표 비율 = 종목별 목표 금액 계산
- 목표 금액 - 현재 보유 금액 = 차이 → 매수/매도/유지 결정
- 차이가 1주 미만이면 유지로 처리 (불필요한 소액 거래 방지)
- 매도 발생 시 안내 문구: 매도 회수 금액을 투자금과 합산해 매수에 활용

#### 2.5.2 액션 유형

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 매수 | 목표 금액 > 현재 보유 금액 — 매수 주수 + 금액 표시 | P0 |
| 매도 | 목표 금액 < 현재 보유 금액 — 매도 주수 + 금액 표시 | P0 |
| 유지 | 차이가 1주(한국) 또는 0.5주(미국) 미만 — 거래 불필요 | P0 |
| 현재 비율 표시 | 현재 비율 vs 목표 비율 괴리(%p)를 색상으로 구분 표시 | P0 |
| 매도 후 안내 | 매도 종목 발생 시 회수 금액 합산 안내 문구 표시 | P0 |
| 리밸런싱 알림 | 목표 비율 괴리 5%p 이상 시 홈 화면에 알림 배지 표시 | P1 |
| 투자 캘린더 | 매월 투자 실행일 설정 및 리마인더 (푸시/이메일) | P2 |
| 수익 현황 조회 | 스냅샷 간 비교로 평가손익·실현손익·총 수익률 자동 계산 및 시각화 | P2 |

### 3.2 핵심 사용자 여정

**① 맛보기 사용자 – 가입 없이 체험:**
1. 랜딩 → '맛보기' 클릭 → 맛보기 모드 진입 (가입 불필요)
2. 갤러리 탐색 → 템플릿 선택 → 상세 패널 확인 → 편집 화면 진입
3. 종목 추가/삭제, 비율 조정, 매수 수량 계산까지 체험 가능
4. '저장하기' 클릭 → 가입 유도 모달 팝업 → 가입 완료 후 편집 내용 그대로 저장

**② 신규 로그인 사용자 – 빠른 시작:**
1. 랜딩 → 로그인 또는 회원가입 → GNB '대가 포트폴리오' 또는 '+ 새로 만들기'
2. 템플릿 갤러리 → 카드 탭 → 상세 패널 확인 → '사용하기' → 편집 화면
3. 월 투자금 입력 → 비율 조정 (선택) → 저장 → 이달의 매수 가이드

**③ 기존 사용자 – 월 루틴:**
1. 앱 접속 → GNB '내 포트폴리오' → 저장된 포트폴리오 선택
2. 이달의 매수·매도 가이드 진입 → 보유 주수·현재가 입력
3. 종목별 매수/매도/유지 액션 확인 → 증권사 앱에서 실행
4. 리밸런싱 알림 발생 시 비율 조정 후 재저장

### 3.3 GNB (글로벌 내비게이션 바)

전체 화면에서 항상 표시되는 상단 메뉴바. 비로그인/맛보기/로그인 상태에 따라 우측 버튼이 달라집니다.

| 상태 | 좌측 | 우측 |
|------|------|------|
| 비로그인 | 로고 + 대가 포트폴리오 | 로그인 \| 시작하기 |
| 맛보기 모드 | 로고 + 대가 포트폴리오 | 맛보기 중 \| 가입하기 |
| 로그인 상태 | 로고 + 대가 포트폴리오 | 내 포트폴리오 \| 프로필 |

맛보기 모드는 회원가입 없이 서비스의 핵심 기능을 체험할 수 있는 게스트 모드입니다.

#### 3.5.1 허용 범위

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 랜딩 페이지 | 전체 열람 가능 | P0 |

| 포트폴리오 편집 | 종목 추가/삭제, 비율 조정, 드래그앤드롭 전부 가능 | P0 |

| 내 포트폴리오 목록 | 불가 — GNB 클릭 시 로그인 화면으로 이동 | P0 |

#### 3.5.2 가입 유도 모달

- **트리거**: 맛보기 모드에서 '저장하기' 버튼 클릭 시
- **모달 구성 요소**:
  - 타이틀: 저장하려면 가입이 필요해요
  - 부제: 가입하면 지금 만든 포트폴리오가 그대로 저장됩니다
  - CTA 버튼 (강조): '가입하고 저장하기'
  - 보조 버튼: '로그인' (이미 계정 있는 경우)
  - 닫기 버튼: 모달 닫고 맛보기 계속
- **가입 완료 후 동작**:
  - sessionStorage에 임시 저장된 포트폴리오 데이터를 서버로 전송
  - 사용자는 편집하던 포트폴리오 그대로 저장 완료 화면으로 이동
  - 데이터 유실 없이 자연스러운 온보딩 경험 제공

#### 3.5.3 임시 데이터 저장 방식

맛보기 중 편집한 포트폴리오는 브라우저 sessionStorage에 JSON 형태로 임시 저장됩니다. 탭을 닫으면 데이터는 소멸합니다.

**처리 흐름:**
1. 편집 이벤트 발생 시 → `sessionStorage['guest_portfolio']`에 실시간 저장
2. 가입/로그인 완료 콜백 → sessionStorage 데이터 존재 여부 확인
3. 데이터 존재 시 → 서버 `POST /portfolios` 호출로 이전
4. 이전 완료 → sessionStorage 초기화

### 5.2 Asset (종목)

| 필드 | 타입 | 설명 |
|------|------|------|
| ticker | string | 종목 코드 (예: AAPL, 005930) |
| color | string (hex) | 티커 해시 기반 고정 색상. 저장 시 확정되며 이후 변경 없음 |

### 5.3 ExecutionRecord (실행 기록)

저장 버튼 클릭 시마다 생성되는 기록입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string (UUID) | 실행 기록 고유 식별자 |
| portfolioId | string | 연결된 포트폴리오 ID |
| executedAt | timestamp | 저장(실행) 일시 |
| totalBudget | number | 해당 월 총 투자금 (원) |
| actions | ActionItem[] | 종목별 액션 배열 (매수/매도/유지) |
| memo | string \| null | 사용자 메모 (예: 3월 적립) |

**ActionItem 구조:**
- `ticker`: string — 종목 코드
- `action`: buy | sell | hold — 액션 유형
- `quantity`: number — 매수(+) 또는 매도(-) 주수
- `pricePerShare`: number — 실행 시점 주당 가격
- `totalAmount`: number — 총 거래 금액 (quantity × pricePerShare)

### 5.4 PortfolioSnapshot (포트폴리오 스냅샷)

저장 버튼 클릭 시마다 생성되는 해당 시점의 포트폴리오 전체 상태입니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string (UUID) | 스냅샷 고유 식별자 |
| portfolioId | string | 연결된 포트폴리오 ID |
| executionRecordId | string | 연결된 실행 기록 ID (1:1 대응) |
| savedAt | timestamp | 스냅샷 생성 일시 (저장 버튼 클릭 시각) |
| assets | SnapshotAsset[] | 해당 시점 전체 종목 구성 배열 |
| totalValue | number \| null | 해당 시점 총 보유 자산 (보유 주수 입력 시 계산) |

**SnapshotAsset 구조:**
- `ticker`: string — 종목 코드
- `name`: string — 종목명
- `ratio`: number — 해당 시점 목표 비중 (%)
- `shares`: number — 해당 시점 보유 주수
- `pricePerShare`: number — 해당 시점 현재가
- `color`: string — 종목 색상 (hex)

### 5.5 저장 시 처리 흐름

저장 버튼 클릭 한 번으로 아래 세 가지가 원자적으로 처리됩니다:

1. **Portfolio 업데이트** — 현재 종목 구성·비중·월 투자금 덮어쓰기
2. **PortfolioSnapshot 생성** — 저장 시점의 전체 상태 스냅샷 신규 레코드 삽입
3. **ExecutionRecord 생성** — 매수·매도·유지 액션 및 금액 신규 레코드 삽입

> ※ 세 작업은 DB 트랜잭션으로 묶어 처리합니다. 하나라도 실패하면 전체 롤백.

### 5.6 종목 색상 시스템

종목마다 고정된 색상을 부여해 편집 화면, 미니 비중 바, 매수·매도 가이드 등 모든 화면에서 동일한 색으로 표시합니다.

#### 5.6.1 티커 해시 기반 색상 결정

```javascript
function getTickerColor(ticker, palette, usedColors) {
  const hash = ticker.split('').reduce((acc, c) => acc + c.charCodeAt(0), 0);
  let idx = hash % palette.length;
  // 충돌 감지: 같은 포트폴리오 내 이미 사용된 색이면 다음 색으로 밀어냄
  while (usedColors.has(palette[idx])) {
    idx = (idx + 1) % palette.length;
  }
  return palette[idx];
}
```

#### 5.6.2 팔레트 구성

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 팔레트 크기 | 20개 색상 — 15종목 이하 포트폴리오에서 겹침 거의 없음 | P0 |
| 색상 간격 | 인접 색상 간 색조(hue) 차이 18° 이상 확보 | P0 |
| 충돌 감지 | 같은 포트폴리오 내 동일 색 발생 시 팔레트에서 다음 색으로 순환 배정 | P0 |
| 색상 저장 | 포트폴리오 저장 시 확정된 색상을 Asset.color 필드에 함께 저장 | P0 |

- 포트폴리오 편집 화면 — 종목 카드 티커 텍스트 색상 및 비중 바
- 내 포트폴리오 목록 — 카드 내 미니 비중 바 세그먼트

- 보유 현황 입력 카드 — 티커 텍스트 및 현재 비율 바

### 5.7 수익 계산 방식

인접한 두 PortfolioSnapshot을 비교해 평가손익과 실현손익을 각각 계산합니다.

#### 5.7.1 평가손익 (미실현손익)

보유 주수는 변하지 않고 가격만 변한 경우의 손익입니다.

**계산식:** `(이번달 현재가 - 지난달 현재가) × 지난달 보유 주수`

**예시:** 삼성전자 지난달 76,500원 × 10주 → 이번달 82,000원 × 10주  
→ 평가손익 = (82,000 - 76,500) × 10 = **+55,000원**

#### 5.7.2 실현손익

ExecutionRecord의 매수·매도 액션에서 직접 계산합니다.
- **매도 실현손익:** (매도가 - 평균 매수가) × 매도 주수
- 평균 매수가는 해당 종목의 과거 ExecutionRecord 매수 기록을 누적 평균해 산출
- 매수 거래는 실현손익 없음 (미래 평균 매수가 산출에만 반영)

#### 5.7.3 표시 지표

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| 총 투자 원금 | 누적 매수 금액 합산 (ExecutionRecord 기준) | P2 |
| 현재 평가금액 | 보유 주수 × 현재가 합산 (최신 스냅샷 기준) | P2 |
| 총 평가손익 | 현재 평가금액 - 총 투자 원금 | P2 |
| 총 수익률 | 총 평가손익 ÷ 총 투자 원금 × 100 (%) | P2 |
| 종목별 수익률 | 종목별 (현재가 - 평균 매수가) ÷ 평균 매수가 × 100 (%) | P2 |
| 월별 수익률 추이 | 스냅샷 간 총 평가금액 변화를 월별 꺾은선 그래프로 표시 | P2 |
| 포트폴리오 간 비교 | 복수 포트폴리오의 수익률을 동일 그래프에서 비교 | P2 |

> ※ 수익 계산은 참고용이며 세금·수수료는 미반영입니다. 실제 손익과 다를 수 있습니다.

## 7. 추가 제안 기능

### 7.1 투자 성향 진단 (온보딩)

신규 사용자 진입 시 5~7개 질문으로 투자 성향을 파악하고 적합한 템플릿을 자동 추천합니다. 질문 결과를 MDD 허용 범위와 매핑해 전략 방식을 추천합니다.

> 예: MDD 10% 이하 선호 → 자산배분 전략, 수익률 극대화 선호 → 가치투자 / 퀀트·팩터 전략

### 7.2 시뮬레이션 – 과거 성과 백테스트

선택한 포트폴리오를 과거 N년 동안 매달 적립했다면 수익이 얼마였는지 시뮬레이션으로 보여줍니다.

### 7.3 환율 자동 반영

한국 사용자가 미국 주식 투자 시 원/달러 환율을 반영하여 원화 기준 투자 금액과 매수 수량을 계산합니다.

### 7.4 포트폴리오 공유

자신이 구성한 포트폴리오를 링크로 공유하거나, 커뮤니티에 공개할 수 있는 기능입니다.

### 7.5 증권사 연동 (장기 로드맵)

KIS(한국투자증권) 오픈API 또는 eTrade API를 통해 실제 매수까지 앱 내에서 처리하는 기능입니다.
